# Run Reports

This directory holds eval reports generated by `scripts/eval_digest.py`.

Reports are named `<YYYY-MM-DD>-<git-short-hash>-<variant>.md`, one per run.
The runner auto-appends a "Drift watch" section comparing against the most
recent prior report for the same variant.

## Generating a report

```bash
just eval                   # run v1 against full labels.jsonl
just eval-dry               # dry-run (no LLM calls; mirrors labels as predictions)
just eval-fixtures          # run against 5-example fixture set

# Or directly:
python3 scripts/eval_digest.py --variant v1
python3 scripts/eval_digest.py --variant v2
```

Requires `ANTHROPIC_API_KEY` in the environment for non-dry-run mode.

## Why this directory may be empty

The labelled set currently contains 18 real examples reconstructed from
the project's Linear ticket history (CC-9 through CC-28). The eval harness
itself is the deliverable; the first live run is pending until either:

1. The label set grows to a more statistically useful size (target ≥50), or
2. A baseline run is generated with the current 18 examples to anchor
   future drift comparisons.

To produce the first baseline, run `just eval` after setting
`ANTHROPIC_API_KEY` — the report will land here.
