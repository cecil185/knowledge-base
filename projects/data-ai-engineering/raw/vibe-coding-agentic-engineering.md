# Vibe coding and agentic engineering are getting closer than I'd like (Simon Willison)

**Source:** https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/
**Read:** 2026-05-18

## The convergence of vibe coding and agentic engineering

Simon Willison previously drew a clear line between "vibe coding" (writing code without understanding or caring about quality — fine for personal throwaway tools) and "agentic engineering" (professionals using AI tools while maintaining full responsibility for security, maintainability, and performance). After a podcast conversation with Joseph Ruscio on Heavybit's High Leverage podcast, he acknowledges these two modes are converging uncomfortably.

## The trust paradox

As coding agents become more reliable, Willison finds himself not reviewing every line of production code. He increasingly treats AI agents like autonomous teams in larger organizations — relying on their output without exhaustive code review, then investigating only when problems emerge.

The critical tension: unlike human teammates, AI agents cannot build professional reputation or accountability. Willison asks: "if I haven't reviewed the code, is it really responsible?" He identifies this as normalization of deviance — each successful unreviewed deployment increases confidence, potentially until failure occurs.

## Evaluating AI-generated work

AI tools now produce repositories with hundreds of commits, comprehensive tests, and polished documentation in hours — indistinguishable from projects representing months of careful human work. Willison's key insight: actual usage history matters more than superficial quality markers. Software that has been actively used for weeks carries more value than freshly generated code, regardless of how polished the documentation looks.

## Shifting development bottlenecks

When code production accelerates from 200 to 2,000 lines daily, the entire software development lifecycle — designed around slower creation — requires reimagining. Design processes that were previously necessary because implementation took months can now become counterproductive, since correction costs drop dramatically when code is cheap to produce.

## Professional future and enterprise adoption

Willison remains confident about the professional future. AI tools amplify existing expertise — domain knowledge enables effective tool use. Software development remains extraordinarily difficult despite automation advances. The realistic future involves professional teams using AI tools to build superior products faster, not widespread vibe coding replacing commercial software.

He references Matthew Yglesias's observation: consumers prefer "professionally managed software companies" using AI assistance over personal coding projects — mirroring the general preference for proven, professionally-maintained solutions over homemade alternatives.

For enterprise adoption specifically, proven multi-company validation over extended periods remains essential. Risk tolerance for untested solutions stays low even when development costs drop.
